What is OopisOS?

OopisOS (v1.0.5.3) is a significantly enhanced JavaScript application simulating a desktop operating system environment entirely within your web browser. It now features a retro-style terminal with advanced capabilities like **tab completion**, **command piping**, and **background process execution**. It retains its persistent hierarchical file system (per user), robust user management (including user removal), and comprehensive session handling. The built-in text editor has been vastly improved with **line numbers, a status bar, multiple view modes (split, edit-only, preview-only), a persistent word wrap toggle, and the ability to export previews as HTML**. Common command-line utilities are expanded, and scripting capabilities are more powerful with argument passing and control commands like `delay` and `check_fail`.

Core Architecture: Modular Power

OopisOS is meticulously engineered with a modular JavaScript design. Specialized "Manager" components interoperate to handle distinct OS functionalities, ensuring high maintainability, clear separation of concerns, and testability. This architecture is fundamental to its robust performance and future extensibility.

Key System Managers

CommandExecutor

The brain of command interpretation: tokenizes input (Lexer), builds command structures including pipelines and background tasks (Parser), and dispatches to specific command handlers.

FileSystemManager

Manages the hierarchical file system, including CRUD operations for files/directories, path resolution, and interactions with IndexedDB.

UserManager

Oversees user accounts: registration, login/logout, user removal, and ensures each user has an isolated file system and session state.

SessionManager

Handles the persistence of user sessions, including automatic state saving on logout/login and manual backup/restore functionality.

EditorManager

Powers the `edit` command, providing a full-featured text editor UI with file mode detection, preview capabilities (Markdown/HTML), user-configurable word wrap, line numbers, status bar, multiple view modes, and preview export.

OutputManager

Controls all output to the terminal display, including command results, error messages, system logs, editor-specific messages, and handles output for backgrounded processes.

TerminalUI

Manages the visual and interactive aspects of the terminal interface, such as the dynamic prompt, input field, and caret positioning.

IndexedDBManager

Abstracts IndexedDB operations, providing a clean API for the FileSystemManager to store and retrieve persistent file system data.

HistoryManager

Tracks executed commands, enabling users to navigate and reuse previous inputs via arrow keys. Supports history clearing.

StorageManager

Provides a unified interface for LocalStorage access, used for user credentials, session states, and editor preferences like word wrap.

ConfirmationManager

Handles user confirmation prompts for critical operations (e.g., `rm`, `reset`, `loadstate`, `removeuser`) to prevent accidental data loss.

TabCompletionManager

Provides context-aware tab completion for commands and file paths, enhancing terminal usability.

This modular approach allows for independent development and testing of components, contributing to the overall stability and maintainability of OopisOS.

File System Operations

Includes `mkdir` (with `-p`), `ls` (with `-l`, `-a`), `touch`, `cat`, `cp` (with `-r`, `-f`), `mv` (with `-f`), `rm` (with `-r`, `-R`, `-f`, `-i`), `pwd`, `cd`, `tree` (with `-L`, `-d`), `find`. Enhanced for more POSIX-like behavior and options.

Scripting & Execution

The `run` command executes scripts with argument passing (`$1`, `$@`, `$#`). Features like `delay` for pacing and the diagnostic `check_fail` command enhance script utility and testability.

User & Session Management

Supports `register` (or `useradd`), `login` (or `su`), `logout`, `whoami`, `removeuser`. `savestate`, `loadstate`, `backup`, `restore`, `savefs`, and `reset` provide comprehensive session and FS control, with data persisted per user.

Enhanced Built-in Editor (`edit`)

A full-screen text editor with `Ctrl+S` (save/exit), `Ctrl+O` (exit no save), `Ctrl+P` (toggle preview). Now features line numbers, a status bar (lines, words, chars, cursor), toggleable view modes (split, edit, preview), persistent word wrap, and HTML preview export.

I/O, Utilities & Shell Features

`echo` for output, robust redirection (`>`, `>>`), command piping (`|`), background processes (`&`), and tab completion. Utilities include `date`, `help`, `clear`, `history` (with `-c`), `export` (file download), `upload` (file upload with overwrite prompt), `shutdown`, `reboot`.

Error Handling & Diagnostics

Provides user-friendly error messages. `check_fail` command aids in script-based testing of failure conditions. Improved parsing for complex command structures.

Data Persistence Strategy

IndexedDB

The primary workhorse for complex data. Used for storing entire user file systems (via `FileSystemsStore` object store). Each user's file/directory structure is serialized as JSON and stored under a unique key (e.g., `fs_Guest`). This allows for larger, structured data to persist reliably across browser sessions.

LocalStorage

Employed for simpler key-value data and user-specific settings:

  • User credentials: `oopisOsUserCredentials` (stores registered usernames).
  • Automatic terminal session states: `oopisOsUserTerminalState_` (saves current path, output, input, history per user).
  • Manual session snapshots: `oopisOsManualUserTerminalState_` (for `savestate`/`loadstate` commands).
  • Editor preferences: `oopisOsEditorWordWrapEnabled` (persists the word wrap toggle state).

This dual-storage approach effectively manages complex file system data (IndexedDB) and simpler session metadata/preferences (LocalStorage) within the browser's capabilities.

OopisOS Command Lifecycle

The processing of any command in OopisOS follows a well-defined, multi-stage lifecycle, meticulously orchestrated by the `CommandExecutor` module. This ensures consistency and predictability, now including support for complex structures like pipes and background tasks.

1. User Input Raw command string from terminal.
2. Lexing Input string broken into tokens (words, operators like '>', '>>', '|', '&', strings) by the Lexer.
3. Parsing Tokens are structured into a command pipeline (segments, arguments, redirection, background flag) by the Parser.
4. Execution The command pipeline is processed segment by segment. Output of one can be input to next (piping). Handled by `CommandExecutor`.
5. Output Final result (success/error, data) is processed by `OutputManager` for display, file redirection, or background process notification.

This structured pipeline ensures consistent command processing, including handling of arguments, redirection, piping, and background execution. It supports robust error handling at each stage and facilitates debugging or future enhancements.
