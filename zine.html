Chapter 1: The Lay of the Land

To understand OopisOS, one must first grasp its philosophy. It is a complete, self-contained, and persistent operating system that lives entirely within the confines of your web browser. Nothing you do here can harm your actual computer. Your files, your user accounts, your command history—all of it is saved locally in your browser's storage, waiting for you when you return. It is the ultimate sandbox, a digital terrarium for the curious and the creative.

The primary interface is the command line, a loving homage to the powerful UNIX shells that run the world's mainframes, but with a friendly, modern sensibility. The prompt—Guest@OopisOs:/>—is your starting point. It tells you who you are (Guest), where you are (the root directory, /), and that the system is ready for your command.

Chapter 2: The Core Lexicon - Navigating the World

Like any new land, OopisOS has its own language. Mastering a few key commands will turn you from a tourist into a native in no time. For a quick reference on any command, simply type help [command_name]. For now, let's learn to walk.

The Holy Trinity of Navigation

  • pwd (Print Working Directory): The "You Are Here" map. It tells you your exact location in the file system's hierarchy.
  • ls (List): Shows you the contents of your current location. This is the most versatile tool for exploration, with powerful flags like -l for a detailed long-form view, -a to see hidden files (those starting with a '.'), and -R to recursively list the contents of all subdirectories.
  • cd [directory] (Change Directory): Your method of transport. Use it to move into other directories. cd .. takes you up one level.
// Using `ls` with the `-l` (long) and `-a` (all) flags to inspect the root directory.
Guest@OopisOs:/> ls -la
total 2 drwxr-xr-x 3 root 4096 Jan 01 12:00 . drwxr-xr-x 3 root 4096 Jan 01 12:00 .. drwxr-xr-x 4 root 4096 Jan 01 12:00 etc drwxr-xr-x 3 root 4096 Jan 01 12:00 home

The output of ls -l is packed with information. From left to right: the file type and permissions (e.g., `drwxr-xr-x`), the owner (`root`), the size, the modification date, and the name. This single command tells you almost everything you need to know about your digital surroundings.

Chapter 3: The Spark of Creation

Navigation is only half the fun. OopisOS truly comes alive when you begin to shape it. The system provides a rich set of tools for creating and manipulating files and directories.

CommandPurpose
mkdir [name]Create a new, empty directory. Use the -p flag to create a whole path at once (e.g., mkdir -p project/assets/images).
touch [filename]Creates a new, empty file or, if it already exists, updates its "last touched" timestamp.
echo "[text]"Prints text to the screen. Combine this with redirection to easily create files with content.
cat [filename]Displays the entire contents of a file on the screen.
edit [filename]Opens the powerful built-in text editor, the system's crown jewel.

The true centerpiece of creation is the edit command. This isn't just a simple text pad. It's a context-aware application that transforms based on the file type. For plain text files or shell scripts (.sh), it's a clean, efficient code editor. But when you edit a Markdown (.md) or HTML (.html) file, it reveals its true power, offering a live preview pane and a formatting toolbar that makes writing structured documents a breeze.

// Using `echo` with the `>` redirection operator to create and write to a file in one step.
Guest@OopisOs:/home/Guest> echo "My new project ideas:" > ideas.txt
Guest@OopisOs:/home/Guest> cat ideas.txt
My new project ideas:

Chapter 4: The Power of the Shell

The OopisOS shell is more than just a command interpreter; it's a complete environment for orchestrating complex tasks. By combining commands, you can achieve results that would be tedious or impossible with individual steps.

"This isn't just a command line; it's a programmable environment for data manipulation."
  • Pipes |: The soul of the shell. The pipe takes the output of the command on its left and uses it as the input for the command on its right. This allows you to build "pipelines" that filter and process information on the fly.
  • Redirection > and >>: Control where your output goes. > will send output to a file, overwriting its contents. >> will append the output to the end of the file, which is perfect for creating log files.
  • Sequencing ;: String multiple commands together on a single line to be executed one after another.
  • Backgrounding &: Add an ampersand to the end of a command to run it in the background. The OS will return your prompt immediately, letting you know when the job is done without making you wait.
// A pipeline in action: Get command history, find lines with 'edit', then count them.
Guest@OopisOs:/> history | grep "edit" | wc -l
3

Chapter 5: The System and the Self

OopisOS provides an unprecedented level of control over the system state and your digital identity. It's designed for both security and flexibility.

A Multi-User Metropolis

You are not alone. With useradd, you can create new user accounts, each with a password-protected home directory. The login, logout, and su commands allow for seamless switching between users. This, combined with the permissions system, creates a true multi-user environment. The superuser, root, has the ultimate authority to manage the system.

The Unforgiving `find`

The `find` command is the system's bloodhound. It can search the entire file system based on a dizzying array of criteria: name (-name "*.txt"), type (-type d for directory), owner (-user Guest), and more. But its true power lies in its ability to execute actions on what it finds. You can print the results (-print), delete them (-delete), or even run another command on each result (-exec).

// Using `find` to locate all `.log` files in the home directory and delete them.
Guest@OopisOs:/> find /home -name "*.log" -delete

Total System Recall

Never fear a catastrophic error again. OopisOS provides a suite of commands for total system control. savestate and loadstate allow for manual session snapshots, while backup creates a downloadable JSON file of your entire file system. Should you need to start over, restore can rebuild your world from that backup file, and the ultimate `reset` command will return the entire OS to its factory-fresh state (after a stern confirmation, of course).
