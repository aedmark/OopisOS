üõ†Ô∏èCAPTAIN'S ORDERS: FROM YOUR BRAIN TO THE MAINFRAME! (Command Lifecycle)

Ever wonder how your brilliant command `give_me_buckazoids` (sadly, not a real command) gets processed?

  1. YOU YELL IT (User Input): You type something. Hopefully coherent.
  2. LEXTRON-9000 DECONSTRUCTS (Lexing): Breaks it into tiny, manageable word-bits.
  3. PARSE-O-TRON ASSEMBLES (Parsing): Tries to build a sensible "command pipeline".
  4. DROIDS SCRAMBLE (Execution): The CommandExecutor droid delegates tasks.
  5. RESULTS (OR EXPLOSIONS) ON SCREEN (Output): You see stuff! Or an error message.

üî¨SELF-DESTRUCT SEQU... I MEAN, DIAGNOSTICS! (The `diag.sh` Script)

To prevent total system meltdown (and ensure the devs didn't drink too much Tang before coding), a `diag.sh` script runs NUMEROUS automated tests. It checks if things that should break do break (`check_fail` is its favorite command), and if things that shouldn't break... well, you get the idea. It's the only thing keeping this rust bucket flying straight (mostly).
